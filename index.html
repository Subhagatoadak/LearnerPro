Step 01

Token Embeddings

Every token starts as a learned vector of 8 values. Similar words begin with similar directions before context changes them.

embedding(token_i) = E[token_i]  ∈ ℝd_model

📖 How token embeddings work

▾

Every word in the vocabulary maps to a fixed-length vector of real numbers. Think of it as a huge look-up table: the model assigns each word a unique address in a high-dimensional space.

  1. A vocabulary list maps word strings → integer IDs.
  2. An embedding matrix E of shape [vocab_size × d_model] stores one row per word.
  3. embedding(word) = E[word_id] — just a row lookup.
  4. During training these rows are learned via back-propagation so semantically similar words end up near each other.

In this demo: d_model = 8, so each word is an 8-number vector.

Step 02

Positional Encoding

The same word in a different position should mean something different. Sinusoids inject order into the sequence.

x_i = embedding_i + position_i  |  PE(pos,2k) = sin(pos/100002k/d)  |  PE(pos,2k+1) = cos(pos/100002k/d)

📖 Why sinusoidal positional encodings?

▾

Self-attention has no built-in notion of order — "cat sat" and "sat cat" would look identical to it. Positional encodings inject position information into each token's vector.

  • Sinusoids of different frequencies let the model represent any absolute position, and differences between positions generalise to sequence lengths longer than seen during training.
  • Even dimensions get sin, odd dimensions get cos.
  • Higher dimensions oscillate more slowly, encoding coarser position information.
  • The model can attend to relative positions via linear combinations of these encodings.

Step 03

Multi-Head Self-Attention

Each head projects tokens into query, key, and value spaces, then decides which words matter for each token.

Attention(Q,K,V) = softmax( QKᵀ / √d_k ) · V  |  Q=XW_Q   K=XW_K   V=XW_V

📖 Scaled dot-product attention — step by step

▾
  1. Project: multiply the input X (seq×d_model) by three weight matrices W_Q, W_K, W_V (each d_model×d_head) to get Q, K, V matrices.
  2. Dot products: compute Q·Kᵀ giving a (seq×seq) score matrix — entry [i,j] is how much token i should attend to token j.
  3. Scale: divide by √d_k to prevent vanishing gradients from large dot-products pushing softmax into saturation.
  4. Softmax: each row becomes a probability distribution over keys.
  5. Aggregate: multiply weights by V to get a weighted average of value vectors.

Multiple heads allow attending to different aspects simultaneously. Their outputs are concatenated and projected back: concat(head₁…headₙ) · W_O.

Step 04

Residual Add + LayerNorm

The model keeps the original signal, adds the attention output, then normalizes each token vector so values stay well-scaled.

y = LayerNorm(x + Attention(x))  |  LayerNorm(v) = (v − μ) / √(σ²+ε)

📖 Residual connections & Layer Normalization

▾

Residual (skip) connection: adds the original input back to the attention output: out = x + Attention(x). This means the attention only has to learn corrections, not full representations, which makes training vastly more stable for deep networks.

Layer Normalization: re-scales each token vector independently so its mean is 0 and standard deviation is 1. Unlike Batch Norm, it operates per-token (not per-feature), making it independent of batch size.

  • Compute mean μ and variance σ² over the d_model dimensions.
  • Normalize: (v[i] − μ) / √(σ²+ε)
  • ε (epsilon) ≈ 1e-5 prevents division by zero.

Step 05

Feed-Forward Network

Each token now passes through the same small MLP independently: expand, apply ReLU, then project back to the model width.

FFN(x) = W₂ · ReLU(W₁x + b₁) + b₂  |  8 → 16 → ReLU → 8

📖 Feed-Forward Network — the per-token MLP

▾

After attention has mixed token representations across the sequence, each token independently passes through a small two-layer MLP.

  1. Expand: W₁ (8×16) projects from d_model=8 to d_ff=16.
  2. Activate: ReLU(x) = max(0, x) zeroes out negative activations, creating sparse feature maps.
  3. Project back: W₂ (16×8) compresses back to d_model=8.
  4. Add biases b₁, b₂ at each layer.

Research suggests FFN layers act as key-value memories — keys activate certain neurons, values store factual associations retrieved at inference.

In production models d_ff is typically 4× larger than d_model (e.g. GPT-3 uses d_model=12288, d_ff=49152).

Step 06

Final Contextual Output

After the second residual and normalization, each token has a context-aware representation that depends on the whole sequence.

output_i = LayerNorm(z_i + FFN(z_i))  |  z = AddNorm₁ output

📖 The full transformer encoder block

▾

One complete transformer encoder block applies two sub-layers, each with a residual connection and layer normalization:

  1. z = LayerNorm(x + MultiHeadAttn(x))
  2. output = LayerNorm(z + FFN(z))

The output is a new sequence of the same shape (seq_len × d_model). It encodes each token's contextual meaning — the same word in different sentences will now have different vectors.

Real models (e.g. BERT, GPT) stack 12–96 of these blocks. Each layer captures increasingly abstract relationships.

Step 07

Next-Word Prediction

A linear "LM head" projects each token's final vector onto the vocabulary, then softmax turns the scores into probabilities. The highest-probability token is the model's predicted next word.

logits = finalOutput · Wlm  |  P(word) = softmax(logits)  |  next = argmax P

Key-Value Caching

During autoregressive generation each new token only needs its own Q, but must attend to all past tokens' K and V. Without caching, those would be recomputed every step — wasting O(N²) work. The cache stores them once.

Knowledge Distillation

A large teacher model trains a smaller student by sharing its soft probability distributions over the vocabulary — not just the single correct label. The soft targets encode relationships between classes that a one-hot label throws away.

Post-Training Quantization

Replace 32-bit floats with low-bit integers to shrink model size and speed up inference. INT8 halves memory vs FP16; INT4 halves again. The key trade-off is reconstruction error: fewer bits = more rounding noise.

q = round( w / scale ) + zero_point  |  w̃ = (q − zero_point) × scale  |  scale = (max−min) / (2bits−1)

LoRA — Low-Rank Adaptation

Instead of updating the full weight matrix W (d×d) during fine-tuning, LoRA injects two small matrices: A (d×r) and B (r×d) with rank r ≪ d. Only A and B are trained; W is frozen. The effective update is ΔW = B·A. Parameters drop from d² to 2·d·r.

W' = W0 + (α/r) · B·A  |  params: d² → 2·d·r  |  savings: (1 − 2r/d) × 100%
